## Decisions

- [[0001-hybrid-ai-insights-engine]] — accepted. Rule-based engine is the numeric source of truth; LLM only phrases.
- [[0002-openrouter-over-openai]] — accepted. Free-tier OpenRouter chat completions, Llama 3.1 8B default, graceful degradation when the key is absent.
- [[0003-team-isolation-via-supabase-rls]] — accepted. Per-`team_id` RLS on every team-scoped table, admin bypass via `is_admin()`, service-role-only access for `henrik_cache` + cron writes.
- [[0004-henrik-proxy-cache-strategy]] — accepted. Server proxy at `/api/henrik/*` + Postgres `henrik_cache` with TTLs (account 1h / matches 10m / MMR 2m / history 15m). Normalization at ingest.

Also-made (non-ADR) decisions:

- Ship the full platform in one pass rather than a phased MVP — the spec was concrete enough that deferring to phases would just add rebasing cost.
- Keep `types/database.ts` as a placeholder `Record<string, unknown>` until the live Supabase project is up and `supabase gen types` can run.

## Gotchas

- **Supabase typed-client `never` inference** — passing the placeholder `Database = Record<string, unknown>` as the `<Database>` generic to `createServerClient` / `createBrowserClient` / `createClient` collapsed every query result to `never`. Fix: strip the generic from all three clients in `lib/supabase/` and type rows at call-sites via `types/domain.ts`. Re-introduce the generic once `supabase gen types` has produced a real schema. Documented here so next session doesn't re-debug it.
- **"Surf'n Bulls" quoting in the SQL seed** — the apostrophe inside the team name has to be SQL-escaped as `'Surf''n Bulls'` in the `0003_seed.sql` insert. Lose the escape and the migration fails silently with a syntax error.
