## Decisions

- Accepted [[0005-private-match-vod-uploads]]:
  - one uploaded MP4 per match
  - private Supabase Storage bucket (`match-vods`)
  - app-issued signed upload/download URLs
  - legacy external `vod_url` remains as fallback
  - no embedded player / transcoding yet
- Used **lazy bucket creation** from the service-role path so the repo-backed app can self-provision the `match-vods` bucket on first use instead of depending on a manual dashboard step.
- Used **Node's built-in `node:test`** for the first helper-level tests instead of introducing Vitest/Playwright before the broader testing decision is made.
